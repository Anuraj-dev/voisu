- did: ran /refine on the GTK-vs-Electron question; dispatched two parallel Sonnet research agents
  (HyprVox repo + web on KWin/Wayland); synthesized a weighted comparison → decisive verdict = keep
  GTK4 + gtk4-layer-shell. Key findings: Electron has NO wlr-layer-shell support on Wayland
  (`setAlwaysOnTop` no-op, positioning/getPosition broken), a 150–400MB Chromium runtime, and needs a
  cross-process JS bridge; HyprVox uses Electron only via forced XWayland self-positioning + Hyprland
  window rules + a rich React/canvas waveform (documented XWayland dependency, ~1 GPU-crash/day, 266MB
  runtime) — none transferable to Voisu. GTK4 route: real `zwlr_layer_shell_v1` surface on KWin,
  default no-keyboard-focus, empty-input-region click-through, ~70KB add-on to already-installed GTK,
  single native Rust process via gtk4-rs (KDE ships `layer-shell-qt`; `gtk4-layer-shell` lists KDE
  Plasma as supported).
- changed: docs/STATE.md (in-progress block), docs/decisions.md (overlay toolkit decision), this session
  log. Vault prompt log appended (Obsidian /refine log).
- decided: keep the Overlay on GTK4 + gtk4-layer-shell; reject an Electron migration (-> decisions.md).
