  (HyprVox repo + web on KWin/Wayland); synthesized a weighted comparison → decisive verdict = keep
  GTK4 + gtk4-layer-shell. Key findings: Electron has NO wlr-layer-shell support on Wayland
  (`setAlwaysOnTop` no-op, positioning/getPosition broken), a 150–400MB Chromium runtime, and needs a

  single native Rust process via gtk4-rs (KDE ships `layer-shell-qt`; `gtk4-layer-shell` lists KDE
  Plasma as supported).
- changed: docs/STATE.md (in-progress block), docs/decisions.md (overlay toolkit decision), this session
