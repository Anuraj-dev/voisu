# Voisu codebase audit — 2026-07-18

Branch `feature/transcription-accuracy` @ `a0899b3`, ~23.7k lines of Rust.
Evidence: full source read (four parallel audit passes), `cargo test --workspace`
(297 passed, 0 failed). Tooling caveat: clippy, cargo-audit, cargo-deny not
installed on this machine — lint/CVE-level evidence absent.

## Verdict

Healthy, unusually disciplined codebase. Single biggest risk: a panic in the
spawned capture pump wedges the daemon in `Processing` forever (unsupervised
`tokio::spawn` + `pump.await.expect`).

## Scorecard

| Dimension | Score | Justification |
|---|---:|---|
| Type safety | 8/10 | Zero unwrap/panic in core non-test code, no lossy audio-math casts, no-`Debug` `Credential`, typed IPC enums — docked for 21 `expect()`s and 4 `unreachable!()`s in the daemon binary. |
| Modularity & scalability | 6/10 | Exemplary core↔app boundary and overlay feature hygiene, but the provider seam is a trait in name only and `system.rs` is a 5,659-line god-module. |

| Testing & e2e | 7/10 | 297 green tests through public interfaces with real HTTP/WS/D-Bus wire protocols; but every real OS surface (PipeWire, compositor/libei, live vendor APIs) is faked or behind one `#[ignore]`d manual smoke test. |

## Findings

### Critical (both verified first-hand by the driver)

- **C1 — unsupervised `capture_pump` panic wedges the daemon permanently.**
  `voisu-daemon.rs:1396`: `pump.await.expect("capture pump should not panic")`
  inside `process_recording`, bare-`tokio::spawn`ed at `:577, 893, 982`. A panic
  in the pump (e.g. any poisoned `PipeWireActiveCapture` mutex,
  `system.rs:1284-1478`) silently kills the task; `ActorMessage::Completed`
  never arrives; `actor_loop` stays in `Processing` and rejects all commands
  until daemon restart. The replay path guards exactly this
  (`supervise_replay`, `voisu-daemon.rs:498-505`). Fix: mirror that supervision.
- **C2 — blocking stop path inside async futures.**
  `PipeWireActiveCapture::finish`/`abort` (`system.rs:1476, 1488`) call
  `stop_child` (`:1358-1417`) whose `wait_for_child` + two `bounded_join`s
  busy-poll with `thread::sleep` up to ~2 s each, directly on a Tokio worker
  thread. Fix: wrap in `spawn_blocking` (file already does this for curl/libei).

### Major

- STT provider seam not pluggable: `enum Provider { Deepgram, Groq }`
  (`voisu-core/lib.rs:425-428`), fixed two-field `ProviderStreams`, hardcoded
  `.deepgram`/`.groq` in `ProviderCoordinator` (`lib.rs:1623-1624, 1651-1652`),
  ~6 daemon take-sites, hardcoded `[Deepgram, Groq]` arrays
  (`voisu-daemon.rs:1528`, `system.rs:785`).
- `system.rs` god-module: 5,659 lines, ten concerns, no internal `mod` split
  (regions at `system.rs:92, 155, 566, 628, 1234, 1524, 1894, 2651, 2836, 3350+`).
- Zero sandboxing in systemd units (`packaging/voisu.service`,
  `voisu-overlay.service`): no `NoNewPrivileges`, `ProtectSystem`,
  `PrivateTmp`, `RestrictAddressFamilies`.
- Real OS integration untested: `PipeWireCapture` (`system.rs:1234-1524`) never
  instantiated by tests (all use `ControlledCapture` via `VOISU_TEST_MODE`);
  portal/libei path (`system.rs:3350-4300`) and GTK overlay
  (`bin/voisu-overlay.rs:174-269`) untested; only one `#[ignore]`d live smoke.
- `actor_loop` ~640 lines (`voisu-daemon.rs:368-1006`) with the
  spawn-process_recording block duplicated 3× (`:575-586, 891-902, 979-991`).
- Dependency set unverified against CVE feeds (cargo-audit not installed);
  pinned versions look current (`rustls 0.23.42`, `zbus 5.17.0`,
  `tokio-tungstenite 0.24`).

### Minor

- 21 non-test `expect()`s (daemon actor take-sites + capture mutex locks);
  4 `unreachable!()` state backstops (`voisu-daemon.rs:566, 883, 971, 2071`).
- Bare `String` in `Command::Export`/`Command::Replay`
  (`voisu-core/lib.rs:76,79`) — no newtype.
- Duplicated portal failure-cleanup triplet in
  `FedoraRemoteDesktopPortal::connect` (`system.rs:3157-3257`).
- `transmute_copy` for dlsym pointers (`system.rs:3655-3676`) — standard but a
  `libloading`-style wrapper would be safer.
- 43 `thread::sleep`s in `daemon_cli_lifecycle.rs`; bare 50 ms sleeps at
  `:272, :350, :787` are CI flake risks.
- Zero doc-tests in both crates.

## What's genuinely good — do not refactor away

- `Credential` with no `Debug` impl and a single named escape hatch.
- Three-layer fail-closed diagnostic redaction (env allowlist + value scrub +
  structural URL strip), 0600/0700 files, atomic O_EXCL writes.
- Verified-clean core↔app boundary; GTK overlay feature isolation.
- Tests through real wire protocols and real spawned binaries (real local
  HTTP/WS servers, private `dbus-daemon`), mocking at process boundaries only.
- libei native text-event injection (transcripts can't become keystrokes);
  stdin-piped subprocesses with `env_clear` + PDEATHSIG.
- Load-bearing comment discipline (reaper semantics, shutdown ordering,
  reconciliation heuristics).

## E2e recommendation (deferred — see map fog)

Nested headless compositor (`kwin_wayland --wayland-only` or sway) + real
`xdg-desktop-portal` + PipeWire null source looping a WAV + existing local fake
Groq/Deepgram servers; drive the real CLI, assert transcript via `wl-paste`,
spawn real `voisu-overlay` and assert the layer-shell surface.
